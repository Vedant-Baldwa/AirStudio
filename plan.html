01

Project Overview

AirStudio is a browser-based, hardware-free music performance platform. Using a standard webcam and computer vision, it tracks a user's hand gestures in real-time and maps those gestures to musical notes, rhythms, and instrument sounds across multiple virtual instruments.

Computer Vision

Hand Tracking

MediaPipe Hands detects 21 3D landmarks per hand at 30fps, running entirely in-browser via WebAssembly. No backend needed for CV processing.

Audio

Sound Synthesis

Tone.js + Web Audio API for low-latency instrument synthesis. Sampled soundbanks for realistic instrument tones with reverb, sustain, and dynamics.

ML

Gesture Classifier

TensorFlow.js gesture classifier trained on hand landmark angles. Identifies chord shapes, strumming velocity, finger positions, and instrument-switch gestures.

Application

React Frontend

React + Canvas for AR overlay rendering. Real-time state management for active instrument, looping, recording, and multiplayer sync.

Core Design Principles

  • Zero hardware required — works on any device with a webcam
  • Sub-50ms latency — gesture to sound must feel instantaneous
  • Progressive complexity — beginner gestures work immediately; advanced gestures unlock more expressive play
  • No installation — runs fully in-browser, no server-side ML inference
  • Instrument fidelity — each instrument has unique gesture semantics, not just reskinned controls

Data Flow per Frame

  1. Webcam frame captured via requestAnimationFrame
  2. Frame fed to MediaPipe Hands — returns NormalizedLandmarkList
  3. Landmark array passed to GestureEngine — returns GestureEvent
  4. GestureEvent dispatched to active InstrumentModule
  5. InstrumentModule returns MusicalEvent (note, velocity, duration)
  6. AudioEngine receives MusicalEvent and schedules playback
  7. Canvas overlay updated with visual feedback on same frame
ℹ️

Key constraint: All processing must complete within one frame (~33ms at 30fps). MediaPipe WASM typically takes 8–15ms. Gesture classification 2–5ms. Audio scheduling is async. Total budget is well within 33ms on modern hardware.

Setup Code

setup.js
import { Hands, HAND_CONNECTIONS } from '@mediapipe/hands';
import { Camera } from '@mediapipe/camera_utils';

const initHandTracking = (videoEl, onResults) => {
  const hands = new Hands({
    locateFile: (file) => `https://cdn.jsdelivr.net/npm/@mediapipe/hands/${file}`
  });

  hands.setOptions({
    maxNumHands: 2,
    modelComplexity: 1,          // 0=lite, 1=full
    minDetectionConfidence: 0.7,
    minTrackingConfidence: 0.6,
    selfieMode: true,             // Mirror for natural feel
  });

  hands.onResults(onResults);

  const camera = new Camera(videoEl, {
    onFrame: async () => {
      await hands.send({ image: videoEl });
    },
    width: 1280, height: 720
  });
  camera.start();
  return { hands, camera };
};

Gesture Classifier (TF.js)

gestureClassifier.js
import * as tf from '@tensorflow/tfjs';

// Feature extraction from 21 landmarks → 42 angle features
const extractFeatures = (landmarks) => {
  const angles = [];
  
  // Compute joint angles for each finger
  const fingerJoints = [
    [1,2,3,4],   // thumb
    [5,6,7,8],   // index
    [9,10,11,12], // middle
    [13,14,15,16], // ring
    [17,18,19,20]  // pinky
  ];
  
  fingerJoints.forEach(joints => {
    for (let i = 0; i < joints.length - 2; i++) {
      const angle = computeAngle(
        landmarks[joints[i]], 
        landmarks[joints[i+1]], 
        landmarks[joints[i+2]]
      );
      angles.push(angle);
    }
  });
  
  return angles; // 15 angles total (3 per finger)
};

const computeAngle = (a, b, c) => {
  const ba = { x: a.x - b.x, y: a.y - b.y };
  const bc = { x: c.x - b.x, y: c.y - b.y };
  const cosAngle = (ba.x*bc.x + ba.y*bc.y) / 
    (Math.sqrt(ba.x**2 + ba.y**2) * Math.sqrt(bc.x**2 + bc.y**2));
  return Math.acos(Math.max(-1, Math.min(1, cosAngle))) * (180 / Math.PI);
};

Velocity & Dynamics

velocityEstimator.js
class VelocityEstimator {
  constructor() {
    this.history = [];
    this.windowSize = 5; // frames
  }
  
  update(wristY) {
    this.history.push({ y: wristY, t: Date.now() });
    if (this.history.length > this.windowSize) this.history.shift();
  }
  
  getVelocity() {
    if (this.history.length < 2) return 0;
    const first = this.history[0];
    const last = this.history[this.history.length - 1];
    const dy = Math.abs(last.y - first.y);
    const dt = (last.t - first.t) / 1000;
    return Math.min(1, dy / dt / 2); // Normalized 0–1
  }
  
  getMidiVelocity() {
    return Math.round(this.getVelocity() * 127); // 0–127 MIDI
  }
}

Audio Engine

Built on Tone.js which provides a Web Audio abstraction layer with precise timing, synthesis, and effects. Each instrument module has its own sampler instance connected to a shared effects chain.

audioEngine.js
import * as Tone from 'tone';

class AudioEngine {
  async init() {
    await Tone.start(); // Must be triggered by user gesture
    
    // Master effects chain
    this.reverb = new Tone.Reverb({ decay: 1.5, wet: 0.2 });
    this.compressor = new Tone.Compressor(-20, 4);
    this.limiter = new Tone.Limiter(-6);
    
    // Route: sampler → compressor → reverb → limiter → output
    Tone.Destination.chain(this.compressor, this.reverb, this.limiter);
    
    // Load all instrument samplers
    this.samplers = {
      guitar: await loadGuitarSampler(),
      drums:  await loadDrumSampler(),
      piano:  await loadPianoSampler(),
      violin: await loadViolinSampler(),
    };
  }

  playNote({ instrument, note, velocity = 0.8, duration = '8n' }) {
    const sampler = this.samplers[instrument];
    sampler.triggerAttackRelease(note, duration, Tone.now(), velocity);
  }

  startNote({ instrument, note, velocity }) {
    this.samplers[instrument].triggerAttack(note, Tone.now(), velocity);
  }

  stopNote({ instrument, note }) {
    this.samplers[instrument].triggerRelease(note, Tone.now());
  }
}

// Guitar sampler (uses real guitar samples from public domain)
const loadGuitarSampler = () => new Promise(resolve => {
  const sampler = new Tone.Sampler({
    urls: {
      'E2': 'guitar-E2.mp3', 'A2': 'guitar-A2.mp3',
      'D3': 'guitar-D3.mp3', 'G3': 'guitar-G3.mp3',
      'B3': 'guitar-B3.mp3', 'E4': 'guitar-E4.mp3',
    },
    baseUrl: '/samples/guitar/',
    onload: () => resolve(sampler.toDestination()),
  });
});

Loop Recorder

loopRecorder.js
class LoopRecorder {
  constructor(audioEngine) {
    this.engine = audioEngine;
    this.loops = [];       // Multiple stacked loops
    this.recording = null; // Active recording
    this.loopLength = 4;   // bars
  }

  startRecording() {
    this.recording = { events: [], startTime: Tone.now() };
    Tone.Transport.start();
  }

  captureEvent(musicalEvent) {
    if (!this.recording) return;
    this.recording.events.push({
      ...musicalEvent,
      time: Tone.now() - this.recording.startTime
    });
  }

  stopAndLoop() {
    const { events, startTime } = this.recording;
    this.recording = null;
    
    const loopDuration = Tone.now() - startTime;
    const part = new Tone.Part((time, event) => {
      this.engine.playNote({ ...event, time });
    }, events.map(e => [e.time, e]));
    
    part.loop = true;
    part.loopEnd = loopDuration;
    part.start();
    this.loops.push(part);
    return part;
  }
}

Instrument Modules

Guitar Module

The guitar is the flagship instrument. The right hand (dominant) strums — the vertical velocity of the wrist triggers a chord. The left hand forms chord shapes using finger configurations.

Left Hand Shape Gesture Description Chord Triggered
Open G Middle + Ring down, others up G Major
Capo C Index bar across, others curl C Major
Open D Index + Middle + Ring angled down D Major
Power 5th Fist + Index extended E5 Power Chord
Open E Fist fully closed E Minor (muted → release)
Am shape Thumb up + 3 fingers curled A Minor 
guitarModule.js
const CHORDS = {
  'G': ['G2','B2','D3','G3','B3','G4'],
  'C': ['C3','E3','G3','C4','E4'],
  'D': ['D3','A3','D4','F#4'],
  'Em': ['E2','B2','E3','G3','B3','E4'],
  'Am': ['A2','E3','A3','C4','E4'],
  'E5': ['E2','B2'], // power chord
};

export class GuitarModule {
  constructor(audioEngine) {
    this.audio = audioEngine;
    this.currentChord = null;
    this.velocityEstimator = new VelocityEstimator();
    this.lastStrumDir = null;
    this.strumThreshold = 0.015; // normalized delta-Y
  }

  processFrame({ leftHand, rightHand }) {
    if (leftHand) {
      this.currentChord = detectChordShape(leftHand.landmarks);
    }

    if (rightHand && this.currentChord) {
      const wristY = rightHand.landmarks[0].y;
      this.velocityEstimator.update(wristY);

      const strumEvent = detectStrum(rightHand.landmarks, this.lastStrumDir);
      if (strumEvent) {
        this.lastStrumDir = strumEvent.direction;
        const velocity = this.velocityEstimator.getVelocity();
        const notes = CHORDS[this.currentChord] || CHORDS['G'];

        // Arpeggiate the chord with slight delay between strings
        notes.forEach((note, i) => {
          const delay = strumEvent.direction === 'down' ? i * 0.015 : (notes.length - i) * 0.015;
          this.audio.playNote({ 
            instrument: 'guitar', note, velocity, 
            duration: '4n', 
            time: Tone.now() + delay 
          });
        });
      }
    }
  }
}

Drums Module

The drum module divides the camera frame into virtual "hit zones." When a hand enters a zone with sufficient velocity, the corresponding drum sound fires. Right hand = right kit, Left hand = left kit.

Zone Position Sound
Kick Bottom center Bass drum
Snare Center left Snare drum
Hi-Hat Top right Hi-hat (closed / open)
Crash Top left Crash cymbal
Tom 1 Upper center-left High tom
Tom 2 Upper center-right Floor tom

Piano Module

An 2-octave virtual keyboard is rendered at the bottom of the screen. Finger tips are tracked and trigger notes when they dip below a threshold Y coordinate (simulating pressing a key). Velocity comes from the speed of the downward motion.

Violin Module

The most expressive instrument. Left hand position on a vertical virtual string fretboard determines pitch (continuous, not discrete). Right hand bow motion (horizontal sweep) determines note onset. This creates a continuous, gliding tone unique to violin.

AR Overlay

The AR overlay renders on a canvas element stacked on top of the video feed. It gives users real-time visual confirmation of what the system is detecting, making it learnable.

AROverlay.jsx
const AROverlay = ({ landmarks, activeInstrument, activeChord, noteParticles }) => {
  const canvasRef = useRef();
  
  useEffect(() => {
    const ctx = canvasRef.current.getContext('2d');
    ctx.clearRect(0, 0, ctx.canvas.width, ctx.canvas.height);

    // 1. Draw hand skeleton
    drawHandSkeleton(ctx, landmarks);

    // 2. Overlay instrument visual
    if (activeInstrument === 'guitar') drawGuitarFrets(ctx, landmarks, activeChord);
    if (activeInstrument === 'drums')  drawDrumZones(ctx);
    if (activeInstrument === 'piano')  drawPianoKeys(ctx, landmarks);
    if (activeInstrument === 'violin') drawViolinStrings(ctx, landmarks);

    // 3. Draw note particles (flying notes when sound plays)
    noteParticles.forEach(p => drawParticle(ctx, p));
    
  }, [landmarks, activeInstrument, noteParticles]);

  return <canvas ref={canvasRef} style={{ position: 'absolute', inset: 0 }} />;
};
